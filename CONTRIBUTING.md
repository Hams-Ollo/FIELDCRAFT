# CONTRIBUTING TO THE ZEN SOLUTIONS ARMA KB

Thank you for wanting to contribute. This library grows through community knowledge — veteran players, experienced milsim operators, WCS regulars, and Arma enthusiasts who know things that need to be written down.

---

## WHAT WE NEED

The highest-value contributions right now:

- **Role guides** for Medic, Logistics, Pilot, and Armor Crew
- **Map intelligence** for New Everon, Glenwood, Serhiivka, Rahhaliyah, Rostov
- **Corrections** to existing content — especially mechanics that have changed with updates
- **Mortar tables** from other COP positions on other maps
- **Tactical lessons** from organized sessions — after-action style

---

## STANDARDS FOR CONTRIBUTIONS

**Be accurate.** If you are not sure, say so. A caveat is better than wrong information.

**Be practical.** Every paragraph should answer "so what?" for the reader. If it does not help someone play better, cut it.

**Be readable.** Write for a player, not a staff officer. No unnecessary jargon. Short sentences. Active voice.

**Attribute doctrine.** If you are referencing real military doctrine, note which FM or ATP it comes from. See `DOCTRINE_SOURCES.md` for the reference list.

**Test it.** If you are writing about a mechanic, verify it still works in the current version before submitting.

---

## DOCUMENT FORMAT

All documents use markdown. Follow this structure:

```markdown
# DOCUMENT TITLE
## Subtitle or category

> Brief one-line description of what this document covers

---

## Section Header

Content here.

### Subsection

More detail.
```

Use tables for comparisons, checklists, and reference data. Use prose for explanations, doctrine, and tactics. Do not use bullet points for things that need explanation — write them out.

---

## HOW TO SUBMIT

1. Fork the repository
2. Create a branch named for your contribution (e.g., `medic-role-guide` or `new-everon-map-intel`)
3. Write or edit your document
4. Submit a pull request with a brief description of what you changed and why
5. A maintainer will review, may request changes, and merge when ready

If you do not know how to use GitHub, reach out on the WCS Discord and we can help get your content in.

---

## WHAT WE WILL NOT MERGE

- Content that violates WCS server rules
- Exploit documentation
- Inaccurate mechanics presented as fact without caveats
- Content copied from other sources without attribution
