> ⚠️ **Spawn sides are random each match** — either team may hold either main base. All tactics in these files are written from a geographic perspective (north-spawn / south-spawn / west-spawn), not faction-specific.

## File Structure (all maps)

Each map file follows the same layout:
1. **⚠️ Codename System** — explanation of rotating magenta names + example session rotation from screenshot
2. **Location Pool** — all named objectives with grid, position, tactical role
3. **Map Overview** — axis of advance, FEBA, terrain split
4. **Key Terrain (OCOKA)** — dominant features, high-value OP positions
5. **Supply Routes** — primary logistics corridors, interdiction priorities
6. **Strategic Priority** — push sequence from spawn perspective
7. **Radio Relay Priorities** — relay chain sequence
8. **Mortar Coverage Notes** — TRP candidates, beaten-zone opportunities

---

## Codename System — How It Works

The magenta objective names visible in-game (e.g., KANSAS, RASSVET, BELUGA) are **randomly assigned radio codewords that rotate every match.** They are NOT fixed to locations.

**Correct procedure:**
1. Spawn in — open the map immediately
2. Identify where you are (Main Base)
3. Cross-reference each codeword to the fixed location name using this guide
4. Use the codeword for all radio comms that session — never the location name
